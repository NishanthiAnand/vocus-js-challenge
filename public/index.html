The API Challenge

Anticipated duration: ~90-120 minutes

Instructions

We've provided a basic Express server with 3 REST endpoints - each which are simply returning a static JSON payload. Uh-oh - anybody can hit these endpoints without being authorized ðŸ˜®

We'd like you to implement JWT authentication against all of the REST endpoints. This will involve:

  • Implementing a new /login endpoint. This only needs to be a mock endpoint and only needs to accept a username. The list of mock usernames is in the users.json file.
  • Successfully signing in should grant a user a refresh token and an access token.
  • Refresh token expiry should be configurable, with a default value of 60 seconds.
  • The user's email and role should be encoded into the JWT.
  • Your implementation of authorization should be compatible with many concurrent sessions.
  • A request with an invalid access token should revoke the associated refresh token.
  • Non-authorized users should get a 401 HTTP error code.

Feel free to use any dependencies as you see fit - just explain why you used them in your readme. We definitely anticipate that you'll need at least a crypto library.

For some more info about JWT with refresh tokens, click here.

Bonus points

  • API is containerized with Docker or similar.
  • Solution is deployed to a hosted environment (Heroku, AWS etc).
  • Solution works in a distributed environment.

The Client-side Challenge

Anticipated duration: ~90-120 minutes

Instructions

Vocus support wants a dashboard to see a brief overview of their customers. You'll need to build an Angular or React app following the provided design.

The team lead has given the following requirements:

  • Client app must be built in the /client directory.
  • You must use the provided API's.
  • Solutions must be implemented in Angular, React or Vanilla JavaScript.
  • A Webpack configuration has been provided for convenience, but using it is optional.
  • You may build your app in the `public/customers.html` file, or run your own web server (ng serve/create-react-app etc).


Design

We've used the Bootstrap design language to create a UI, and the Bootstrap CSS bundle has already been included on the page. Clicking on the user card should toggle the visiblity of the users details and dismiss any other cards which are expanded.
